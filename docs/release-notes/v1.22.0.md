## Why these notes are intentionally minimal (dark ship)

v1.22.0's headline work is the P0 Swarm View ("Mission Control for the
agent swarm"). The entire feature ships **DARK** — `swarm_enabled`
defaults `false`, a single server-flippable master gate. With the flag
off, the App Store binary behaves **identically to the already-approved
v1.21.0** for every end user: no Swarm tab, no Live Activity, no new
on-screen surface. H-F1 (provider-spawner refactor + Aider/OpenCode/
Cursor) is **helper-only** and ships through the separate helper `.pkg`
channel, not this binary. D7 i18n localizes the (still dark) swarm
strings.

So the App Store "What's New" must NOT advertise the Swarm View — users
cannot see it in this build. It is honest, modest "stability +
groundwork" copy. The purpose of shipping the dark binary now is to get
the swarm code + the new `NSSupportsLiveActivities` capability through
review, so the later coordinated enable (a server-side flag flip) lights
the feature up with **no second App Store review**.

## App Review Information — reviewer notes (both platforms)

Plain prose, no backticks. Set on the iOS and macOS 1.22.0 versions
before submission. Rationale: the new `NSSupportsLiveActivities`
capability is the one thing a reviewer might probe; the feature is dark,
so preempt the "where is the Live Activity?" question.

```
This update adds the NSSupportsLiveActivities capability. In v1.22.0 the Live Activity is driven entirely by local app state — there is no remote or APNs push involved. The related feature is gated behind a server-side configuration flag and is not user-reachable in this build; it will be enabled later through a staged server rollout, with no further binary changes. APNs-push-driven Live Activities are a documented follow-up planned for a future v1.22.x update. No demo account, special hardware, or extra steps are required to review this build: externally it behaves the same as v1.21.0, which is already approved. Thank you for the review.
```
